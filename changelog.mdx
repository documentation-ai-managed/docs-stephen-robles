## Migration Guide

When updating to a new version, follow these steps to avoid disruptions.

<Steps>
  <Step title="Backup Your Data" icon="database">
    Export all projects via the dashboard.

````bash
# From CLI
npx stephen-robles-docs export --all > backup.json
````
  </Step>

  <Step title="Review Breaking Changes" icon="alert-triangle">
    Check the `Breaking Changes` section in each release above.
  </Step>

  <Step title="Update Configuration" icon="settings">
    <Tabs>
      <Tab title="YAML Config" icon="file">
````yaml
# docs.config.yaml
version: "1.2.0"
brandColor: "#219C50"
features:
  collaboration: true
````
      </Tab>
      <Tab title="Environment Vars" icon="terminal">
````bash
export DOCS_VERSION=1.2.0
export BRAND_COLOR=#219C50
````
      </Tab>
    </Tabs>
  </Step>

  <Step title="Test Locally" icon="play">
    Run your docs site in development mode to verify.
  </Step>

  <Step title="Deploy" icon="upload">
    Push changes to production.
  </Step>
</Steps>

<Callout kind="tip" title="Pro Tip">
  Use the [Stephen Robles CLI](https://docs.example.com/cli) for automated migrations: `npx stephen-robles-docs migrate v1.2.0`.
</Callout>
